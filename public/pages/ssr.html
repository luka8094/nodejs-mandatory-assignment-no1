SSR

Client Side Rendering

Client Side Rendering (CSR) is when the server responds with delivering the individual files, that make up the website's entirety, upon a given fetch request. First the servers dispatches the base document object model (DOM) structure, meaning the HTML then the styling (CSS) followed by the functionality (JavaScript). All these will be rendered in a user's (client) browser.

In other words, the user's browser is responsible for rendering a given website and its content.

Server Side Rendering

Contrary to CSR Server Side Rendering the website gets rendered in full on the server before it gets dispatched to the client browser.

CSR vs. SSR

Each type of rendering has its caveats. They both comes with a set of advantages and disadvantages. The choice will depend on what kind of solution you're aiming for. Here's some lists of a few points to keep in mind:

CSR

  • +

    Fast loading

  • +

    Doesn't need to load the full page

  • +

    Good for dynamic websites and webapps

  • +

    High user traffic

  • -

    Could result in poor User experience

  • -

    Could impact SEO negatively

  • -

    Caching could become an issue

SSR

  • +

    Great for static websites

  • +

    Useful for websites with little interactivity

  • +

    Better choice if SEO is a priority

  • -

    The website needs to be rebuilt every time

  • -

    Problems with high user traffic

  • -

    Expensive to host

  • -

    May depend on internet speed

Generally, it comes down your choice. Neither approach is withous flaws and viable. It will largely depend on what you want your application to achieve for the end user. Various frameworks such as Angular.js, BackBone.js, and React.js are primarily suited for CSR. While other frameworks like Nuxt.js and NEXT.js are developed with a SSR approach in mind.
