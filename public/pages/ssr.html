Client Side Rendering

Client Side Rendering (CSR) is when the server responds with the individual web files on a given request. First it will dispatch the document object model (DOM) structure meaning the HTML markup, followed by the styling and then functionality. All these will be rendered in a user's (client) browser.

In other words, the user's browser is responsible for rendering a given website and its content.

Server Side Rendering

Contrary to CSR Server Side Rendering the website gets rendered in full on the server before it gets dispatched to the client browser.

CSR VS. SSR

There are various advantages and disadvantages to both CSR and SSR. Ultimately, it will depend on what kind of solution you're aiming for.
