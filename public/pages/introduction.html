Variables in Javascript

Lets handle some fundamental Javascript syntax first.

In Javascript you can declare variable through three different keywords: var, let, and const.

The keyword let has a restrained block specific local scope. This means its only accessible within the boundaries of where the variable been declared. In other words, the variable will remain only known to area of a given code block. The value of a let variable can be modifed on the fly. In other words, the value is mutable.

However, the const keyword is used to declare and hold values that can not be overwritten or be Modified. However, the value itself can serve as a base in combination with other variables. Its practical to contain values you'd consider read-only. This phenomenon is known as immutability.

Its important to note that these keywords serve different purposes. Its common practice to avoid using var keyword and substitute it with let and const whenever possible despite having var at your disposal. A reason being is that its an outdated approach and the scope of the keyword can cause a domino effect of complications and difficulties with code maintainability, transparency and tracibility.

A common error is to delcare a loop incrementer with thevar keyword. The accesibility of the variable still breaches the boundary of the loop.

In case of loops, its advised to use always let over var when iterating over collections, Like for example an array.

The upper should be avoided example should in for or while-loops.

All the keywords mentioned here are capable of holding all the primitive data types: string, numeric values, and boolean logic. Additionally, Javascript variables can contain custom defined objects, functions, and data structures as well.

Arrow functions

In the prior section the common way of defining functions was described. However, in some circumstance we can write more brief functions called arrow or, also known as, anonymous functions. Alternatively, they're also known as a Lambda expressions in other languages (derived from the mathematics of Lambda calculus).

This logic stems from the concept of Referential transparency, which essentially means "replacing an expression with something of equal meaning". With this concept we can condense the code syntax and be more concise in how we express sequences of logic.

Callbacks

A callback is a "function reference" and a way for which functions can take other function as an arguments.
